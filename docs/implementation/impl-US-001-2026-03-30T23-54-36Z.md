## 🎯 Objective

Remove runtime imports from external conventional commitlint packages, keep the package contract as an `extends`-only consumer setup, and fix the repository hygiene and workflow reproducibility issues identified in review.

---

## 🧱 Changes

- File: src/conventional-config.ts
  - Added a local conventional ruleset and prompt metadata so the package no longer depends on `@commitlint/config-conventional` at runtime.
- File: src/parser-preset.ts
  - Reworked the parser preset to build emoji-aware header patterns from the local conventional type metadata.
- File: src/index.ts
  - Switched the exported shareable config to a self-contained rules and prompt configuration.
- File: src/index.spec.ts
  - Updated coverage to assert the package exports local conventional rules instead of extending an external package.
- File: package.json
  - Removed external runtime config packages, declared workflow CLIs, and refreshed the dependency model.
- File: .github/workflows/commitlint.yml
  - Replaced transient `npx` calls with `npm exec` so CI uses declared dependencies.
- File: .github/workflows/release.yml
  - Replaced transient `npx semantic-release` usage with `npm exec`.
- File: .gitignore
  - Added ignores for generated dependency, build, coverage, and validation artifacts.
- File: .github/copilot-instructions.md
  - Updated repository guidance to describe the self-contained package architecture.

## 🧪 Notes

- Edge cases handled: supported emoji headers, breaking-change headers, unsupported emoji prefixes, missing emoji prefixes, invalid empty subjects, and header length enforcement.
- Reproduction/regression checks: confirmed the local `extends`-only consumer config still loads and lints correctly through `@commitlint/load` and `@commitlint/lint`.
- Assumptions: local conventional rule and prompt parity is sufficient for this package baseline without depending on external conventional config packages.
- Limitations: this package now owns its conventional metadata locally, so future upstream conventional type metadata changes would need to be updated here intentionally.
- Validation performed: `npm run typecheck`, `npm test`, `npm run validate`, `npm run validate`.
