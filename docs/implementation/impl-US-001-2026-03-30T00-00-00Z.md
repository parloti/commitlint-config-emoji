## 🎯 Objective

Implement the smallest real package that enforces emoji-prefixed conventional commits, supports consumer adoption through a single `extends` entry, and aligns copied repo metadata with the actual commitlint package.

---

## 🧱 Changes

- File: package.json
  - Added the package contract, scripts, and commitlint/Vitest/TypeScript dependencies.
- File: src/index.ts
  - Exported the shareable config that extends `@commitlint/config-conventional`, requires emoji-prefixed headers via a custom parser preset, and enables prompt emojis in the header.
- File: src/parser-preset.ts
  - Added the parser preset builder that rewrites the conventional header patterns to require a supported emoji prefix.
- File: src/index.spec.ts
  - Added executable regression tests for valid emoji headers, missing emoji failures, unsupported emoji failures, header length checks, local consumer setup, and repo guidance drift.
- File: commitlint.config.ts
  - Reduced the local consumer config to `export default {extends: ['@codeperfect/commitlint-config-emoji']};`.
- File: .github/copilot-instructions.md
  - Replaced copied ESLint guidance with package-specific commitlint guidance.
- File: .github/workflows/ci.yml
  - Updated CI to build the package instead of generating unrelated docs.
- File: .github/workflows/release.yml
  - Updated release to build the package instead of generating unrelated docs.
- File: .github/workflows/docs.yml
  - Removed the copied docs deployment workflow because it does not belong to this package baseline.
