## ⚠️ Issues

- [ ] Runtime dependencies are misclassified as dev-only, which will break consumers installing the published package. Severity: High. The shareable config imports `@commitlint/config-conventional` and `conventional-changelog-conventionalcommits` at runtime in [src/index.ts](../src/index.ts#L3) and [src/parser-preset.ts](../src/parser-preset.ts#L3-L4), but both packages are only listed in `devDependencies` in [package.json](../package.json#L29-L40). After publish, consumers extending `@codeperfect/commitlint-config-emoji` will not get those transitive runtime modules unless they install them manually, which violates the story’s `extends`-only adoption contract.

- [ ] The repository is missing a root `.gitignore`, and the current worktree is already polluted with `node_modules` churn. Severity: Medium. This is visible in the current git state where installed dependency files under `node_modules` are showing as untracked changes. That creates a real merge and release hygiene risk for a new package repo and makes accidental vendor commits likely.

- [ ] CI and release workflows rely on `npx`-fetched tooling instead of declared project dependencies. Severity: Medium. [commitlint.yml](../.github/workflows/commitlint.yml#L27-L39) runs `npx commitlint`, and [release.yml](../.github/workflows/release.yml#L57-L60) runs `npx semantic-release`, but neither `@commitlint/cli` nor `semantic-release` is declared in [package.json](../package.json). The workflows may still pass by downloading latest packages at runtime, but that makes the pipeline non-reproducible and decouples CI behavior from the repo’s lockfile.

---

## 💡 Suggestions

- Move runtime-required packages to `dependencies`, keep test/build-only packages in `devDependencies`, and verify the published package works in a clean consumer fixture.
- Add a root `.gitignore` before any further package work so `node_modules`, `dist`, and coverage artifacts do not enter the review surface.
- Declare workflow-critical CLIs in the repo and invoke them through the installed dependency set so CI matches local development and lockfile resolution.
